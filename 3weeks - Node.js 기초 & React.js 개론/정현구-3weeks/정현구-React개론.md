## React란?
> Meta에서 개발한 오픈소스 Javascript 라이브러리 

- **대규모 웹 서비스 UI를 더 편하게 개발하기 위한 기술**

## React의 기술적인 특징 

### 1. 컴포넌트를 기반으로 UI를 표현 

> `컴포넌트`: 화면을 구성하는 요소, UI를 구성하는 요소를 말함 

- 재사용성: 버튼, 네비게이션 바, 입력창 등을 각각의 독립적인 컴포넌트로 만든다. 이러한 특징은 한번 잘 만들면 다른 서비스의 여러 곳에서 재사용 가능 
- 유지보수 용이: 특정 UI에 문제가 생겼을 때 전체 코드를 볼 필요 없이, 해당 컴포넌트 파일만 수정 할 수 있다.
- 가독성: 전체 페이지를 하나의 거대한 HTML 파일로 보는 것이 아니라, `</header>`, `</Contents>`, `</footer>`와 같이 직관적인 구조로 코드를 파악할 수 있다.

결론: 코드를 반복해 입력할 필요 없이 Component만 import하여 사용하면 된다는 간편함이 있으며, 어플리케이션이 복잡해지더라도 코드의 유지보수, 관리가 용이해진다.

### 2. 화면 업데이트 구현이 쉽다.

> `업데이트`: 사용자의 행동(클릭,드래그 등)에 따라 웹페이지가 스스로 모습을 바꿔 상호작용하는 것

React는 복잡한 절차를 일일이 명령하지 않고, 결과만 명시하는 `선언형 프로그래밍` 방식을 취함.
- 명령형 vs 선연형 
    - 명령형: 목적을 이루기 위한 모든 과정을 일일이 설명함. (코드가 간결함)
    - 선언형: 과정은 생략하고 목적을 간결하게 명시함. (코드가 길고 복잡함)
- State(상태) 기반 업데이트 
    - 개발자가 화면을 어떻게 바꿀 지 일일이 코딩할 필요가 없음. 특정 변수인 state 값을 바꾸는 것만으로도 화면이 자동으로 업데이트됨.
    - 예시 - `좋아요 버튼 눌렀을 때 변화`   

    **[전통적인 자바스크립트 방식]**
    ```
    1. 버튼 클릭 이벤트가 발생함.
    2. 개발자가 직접 HTML 요소(DOM)를 찾음 (document.getElementById).
    3. 기존 숫자가 얼마인지 가져옴.
    4. 거기에 +1을 계산함.
    5. 버튼의 텍스트를 새로운 숫자로 직접 갈아 끼움.
    6. 아이콘 색상도 직접 바꿈.

    - 문제점: 화면이 복잡해질수록 개발자가 하나하나 챙기다가 실수할 확률이 높음.
    ```
    **[React의 state 방식]**
    ```
    1. likeCount라는 State(상태) 변수를 만듦 (초기값 0).
    2. UI를 설계할 때 "화면에는 {likeCount}를 보여줘"라고 딱 한 번만 적어둠.
    3. 버튼을 누르면 likeCount 값을 1로 바꿈.
    4. React가 값이 바뀐 것을 감지하고, 화면에서 숫자가 들어가는 부분을 알아서 다시 그려줌.
    ```

### 3. 화면 업데이트가 빠르게 처리된다. 

>DOM (Document Object Model) 이란?
>>브라우저가 HTML 파일을 받아서 화면에 그리기 위해 내부적으로 만드는 **'웹 페이지의 청사진'**
>>>- 정의: HTML 태그들을 나무 가지처럼 연결된 구조(Tree 구조)로 바꾼 객체 모델임.
>>>- 특징: JavaScript는 이 DOM을 조작해서 화면을 바꿈.
>>>- 문제점: 아주 작은 부분만 수정해도 브라우저는 화면 전체의 배치를 다시 계산(Reflow)하고 다시 그리는(Repaint) 과정을 거쳐야 함. 이 과정은 연산량이 많아 매우 느리고 성능을 떨어뜨림

#### Javascript 브라우저 렌더링 5단계 (CRP)
1. DOM 트리 생성: HTML 파일을 읽어서 태그 간의 부모-자식 관계를 나타내는 나무 형태의 DOM을 만듦.
2. CSSOM 생성: CSS 파일을 읽어서 스타일 정보가 담긴 CSSOM을 만듦
3. Render Tree(렌더 트리) 생성: DOM과 CSSOM을 결합하여, 화면에 실제로 보일 요소들만 모은 **'웹 페이지의 청사진'**을 만듦.
4. Layout(Reflow): 각 요소가 화면의 어느 위치에, 어떤 크기로 배치될지 계산하는 '배치 잡기' 작업임.
5. Painting(Repaint): 계산된 위치에 실제 색을 입히고 텍스트를 써서 화면에 그려내는 마지막 과정임.

#### 업데이트가 느려지는 이유 
> 우리가 자바스크립트로 화면을 바꿀 때(DOM 수정), 브라우저는 4번(Layout)과 5번(Painting) 과정을 다시 수행해야 함.
- Reflow (Layout 다시 하기): 요소의 크기나 위치가 바뀌면 발생하며, 연산량이 매우 많음.
- Repaint (Painting 다시 하기): 색상만 바뀌는 등 레이아웃에 영향이 없을 때 발생하지만, 여전히 비용이 드는 작업임.

#### 그로 인해 사용하는 `Virtual DOM`
> virtual DOM이란?
>> 실제 DOM을 조작하기 전에 메모리에서 먼저 실행해 보는 **'일종의 복제판'**'
- 작동 원리: 업데이트가 발생하면 실제 화면(DOM)을 바로 건드리지 않고, 가상의 복제판에 먼저 반영해 봄.
- 집괄 처리(Batch Update): 동시에 여러 업데이트가 발생하면 이를 가상 돔에서 다 모았다가, 최종 결과만 실제 DOM에 딱 한 번 전달함.
- 장점: 브라우저가 가장 힘들어하는 '다시 그리기' 횟수를 획기적으로 줄여줌

[DOM을 직접 수정할 때 (나쁜 예시)]
```html
<script>
  function onClick() {
    const $ul = document.getElementById("ul");
    for (let i = 0; i < 3000; i++) {
      $ul.innerHTML += `<li>${i}</li>`;
    }
  }
</script>
<body>
  <button onclick="onClick()">리스트 추가하기</button>
  <ul id="ul"></ul>
</body>
```
- 과정: for문을 돌면서 리스트 하나를 만들 때마다 DOM에 바로 추가함.
- 결과: 브라우저가 리스트 하나당 한 번씩, 총 3,000번을 새로 그림.
- 성능: 약 4,500ms 소요

[Virtual DOM방식 (좋은 예시)]
```html
<script>
  function onClick() {
    const $ul = document.getElementById("ul");
    let list = "";
    for (let i = 0; i < 3000; i ++) {
      list += `<li>${i}</li>`;
    }
    $ul.innerHTML = list;
  }
</script>
<body>
  <button onclick="onClick()">리스트 추가하기</button>
  <ul id="ul"></ul>
</body>
```
- 과정: 가상 돔(혹은 메모리상의 변수)에 3,000개의 리스트를 먼저 다 만듦.
- 결과: 3,000개가 다 완성되면 실제 DOM에 단 1회만 반영함.
- 성능: 약 250ms 소요 (약 22배 빠름)

## React App 생성하기

### React App(Application)이란?
- 정의: React로 만든 웹 서비스를 보통 React App 또는 React Application이라고 부름.
- 구성: React 라이브러리를 기반으로 돌아가는 하나의 독립된 프로그램 단위임.

### 생성 방법의 종류 
- 수동 설정: Node.js 패키지 생성 → React 설치 → 기타 도구(Webpack, Babel 등) 설정을 직접 함. 다만, 입문자에게는 설정이 매우 복잡하여 권장하지 않음.
- 개발 툴 이용 (Vite): 기본 설정이 모두 적용된 상태로 앱을 생성해 주는 차세대 프론트엔드 개발 툴임

### Vite를 이용한 생성
- 생성 및 구동 순서
    1. 패키지 생성 명령: 터미널에 `npm create vite@latest`를 입력함.   
    2. 프로젝트 설정: 프로젝트 이름을 정하고 프레임워크로 **React**를 선택함.
    3. 서버 가동: 생성된 폴더로 이동하여 `npm run dev` 명령어를 입력해 리액트 웹 서버를 실행함.
    4. 접속: 터미널에 표시된 주소(보통 http://localhost:5173/)를 브라우저에 입력하여 접속함.

### 왜 Vite를 쓰는지?
- 환경 설정 자동화: 원래는 브라우저가 최신 자바스크립트 문법을 이해하도록 돕는 도구들을 일일이 세팅해야 하는데, Vite가 이 과정을 자동으로 해줌.
- Localhost의 의미: 주소에 나오는 localhost는 외부 서버가 아니라 '내 컴퓨터' 자체를 의미함. 5173은 내 컴퓨터 안에서 리액트 서버가 사용하는 전용 포트 번호임.
